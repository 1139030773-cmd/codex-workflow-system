# 新手引导

把用户的模糊想法变成可执行任务，并选择合适的后续技能。

## 工作方式

每轮最多问 3 个问题。问题要短，选项要清楚。用户回答不清楚时，先做保守判断，再继续推进。

## 第一步：判断任务类型

把用户目标归类为以下之一：

- 产品 / 项目开发 → 建议调用 `/project-master`
- 学习技能 → 建议调用 `/learning-coach`
- 修问题 / 报错 → 建议调用 `/debug-fixer`
- 调研 / 决策 → 按研究任务处理，必要时联网核对
- 写作 / 内容 → 明确读者、目的、结构、语气和完成标准
- 混乱 / 跑偏 → 建议调用 `/drift-auditor`
- 阶段结束 / 开新对话前 → 建议调用 `/phase-closeout`

## 第二步：最少问题

只问足够启动的问题：

1. 你最终想得到什么？
2. 这是一次性任务，还是长期项目 / 长期学习？
3. 你现在有什么材料、文件、想法或报错？

## 第三步：输出启动卡片

输出：
- 任务类型
- 当前目标
- 下一步最小行动
- 需要使用的文件
- 是否需要决策闸门
- 建议调用的下一个技能

## 第四步：建立真相源

如果是长期项目或长期学习，建议建立或更新：
- `PROJECT.md`
- `TASK_QUEUE.md`
- `DECISIONS.md`
- `STATE_SNAPSHOT.md`

规则：只设置一个正在进行的任务。

## 禁止事项

- 不要一次问很多问题
- 不要直接跳到大方案
- 不要让用户先学习整套系统
- 不要在没确认前做长期架构决策

## 系统关联

本技能是工作流系统的入口点。完成任务分类后，应主动建议用户切换到对应的子技能继续推进。
