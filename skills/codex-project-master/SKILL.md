# Codex 项目总控

目标：把项目保持在一个清晰、可执行、可验证的状态。

## 开始前

优先读取当前工作区的：

- `PROJECT.md`
- `TASK_QUEUE.md`
- `DECISIONS.md`
- `STATE_SNAPSHOT.md`

如果文件不存在，建议创建。不要依赖长聊天历史作为唯一依据。

## 工作流程

1. 明确项目目标
2. 明确当前阶段
3. 明确成功标准
4. 设置一个 active task
5. 最多保留三个 pending task
6. 判断是否涉及决策闸门
7. 给出下一步行动

## 决策闸门

以下事项必须先说明影响并等待确认：

- 架构变化
- 接口变化
- 数据结构变化
- 目录结构变化
- 新依赖
- 状态管理变化
- 权限 / 登录 / 支付等核心逻辑变化

必须输出：

- 想改什么
- 为什么改
- 影响范围
- 风险
- 是否需要写入 `DECISIONS.md`

## 输出格式

输出一张项目行动卡：

- 当前目标
- 当前阶段
- active task
- 完成标准
- 风险
- 下一步

## 禁止事项

- 不要同时推进多个 active task。
- 不要直接写代码，除非用户明确要求进入执行。
- 不要把规划对话污染成功能实现对话。
