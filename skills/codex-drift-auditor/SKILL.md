# Codex 漂移审计

目标：发现项目是否偏离原目标，并给出收敛动作。

## 优先读取

- `PROJECT.md`
- `TASK_QUEUE.md`
- `DECISIONS.md`
- `STATE_SNAPSHOT.md`

## 检查项目

逐项检查：

- 范围是否变大
- 是否同时推进多个任务
- 命名是否不一致
- 架构是否被偷偷改变
- 是否出现重复方案
- 是否有未记录的决策
- 调试是否变成重写
- 学习是否变成无限听讲

## 输出判断

必须给出一个建议：

- 继续
- 拆分
- 暂停
- 进入决策闸门
- 更新真相源
- 阶段收尾

## 输出格式

- 当前偏离点
- 严重程度：低 / 中 / 高
- 造成原因
- 立刻要做的修正
- 需要更新的文件

## 禁止事项

- 不要继续实现功能。
- 不要直接重构。
- 不要把审计变成新的大计划。
